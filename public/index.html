Use Case Modeling

What is a use case?

A use case is unit of work performed by a system that an actor uses. Each use case accomplishes a user goal. The complete set of use cases represents the entire functionality of that system.

Good Examples

  • Login
  • Order item
  • Withdraw money

Bad Examples

  • Type in password
  • Click on item
  • Enter PIN

What is an actor?

An actor is any person or other external entity that initiates a use case for the system.

Good Examples

  • Student
  • Customer
  • Bank

Bad Examples

  • The system itself
  • Anything completely unrelated to the system

What is a use case diagram?

A use case diagram is a graphical representation of all the use cases in a given system. The diagram shows all the actors and use cases for the system and uses arrows to show the relationships among the actors and use cases.

Learn how to create use case diagrams.

What is a use case specification?

A use case specification is a form of use case documentation. The specification provides additional details about the use case beyond its name. Depending on its level of detail, a use case specification may only be a few sentences or it could be several paragraphs worth of information organized in a large table. Each use case in a system should have its own specification.
